# Docs 索引模板

本文件合并了 `design-docs/index.md` 和 `product-specs/index.md` 的模板——两者结构相似,只在初始化时一起使用。

---

## 设计文档索引 (`docs/design-docs/index.md`)

记录每篇设计文档的位置、当前校验状态和最近校验日期。`doc-gardener` agent 会比对这个索引和实际文档/代码现状。

| 文档 | 主题 | 校验状态 | 最近校验日期 |
|---|---|---|---|
| `core-beliefs.md` | agent-first 的核心运作信念 | ✅ 已校验 | <YYYY-MM-DD> |
| <补充其他设计文档> | | | |

校验状态说明:
- ✅ 已校验:最近确认过和代码现状一致
- ⚠️ 待校验:超过约定周期未确认,可能已过期
- ❌ 已知过期:确认和代码现状不一致,等待修复

---

## 产品规格索引 (`docs/product-specs/index.md`)

记录产品功能规格文档的位置,作为 agent 理解"要做什么、给谁用"的入口。

| 规格 | 一句话描述 | 状态 |
|---|---|---|
| <例如 new-user-onboarding.md> | <一句话描述> | draft / shipped / deprecated |

> 规格文档本身建议保持精简——详细的交互细节、视觉稿,如果存在于仓库外(设计工具等),至少在这里留一句"现状摘要 + 在哪能找到完整版",避免 agent 完全看不到这部分上下文。
