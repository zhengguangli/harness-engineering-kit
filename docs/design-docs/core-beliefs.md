# 核心信念(Core Beliefs)

定义这个仓库里 agent-first 的运作原则。这些是被反复确认过、值得长期保留的判断,不是临时的偏好。

1. **看不见就不存在**:任何只存在于聊天记录/口头共识里的知识,对 agent 等于不存在。新的架构共识、产品决策,必须落进对应的 `docs/` 文件才算"生效"。
2. **约束不变量,不管实现细节**:架构边界和目录结构要被机械强制,边界内部的具体写法允许自由。本仓库的 skill/agent 目录结构就是这条信念的自举示范。
3. **计划是工件,不是草稿**:复杂任务的计划要落盘、版本化、可被后续 agent 接力读取。
4. **失败时先问"缺了什么能力"**:不要把"再试一次"当作默认的修复手段——是否缺少某个 skill、模板、提示词,或者 context 里被它忽略的信息?
5. **熵增需要持续清扫,不要攒成大扫除**:技术债是利息很高的贷款,小额持续还款比集中处理划算。
6. **地图不是百科全书**:CLAUDE.md/AGENTS.md 只放导航和极少数全局硬约束,所有深度内容下沉到 `docs/`。
7. **自举**:本仓库用自己定义的技能(`harness-repo-map`、`harness-architecture-boundaries` 等)来管理自身——如果某条规则在你的仓库里有效但本仓库没遵守,那就是一个需要修复的 gap。
