# 产品规格索引

记录产品功能规格文档的位置,作为 agent 理解"要做什么、给谁用"的入口。

| 规格 | 一句话描述 | 状态 |
|---|---|---|
| README.md(仓库根目录) | 项目的 GitHub 着陆页,面向人类读者:介绍、来源、安装方式、使用流程 | shipped |

本仓库的产品定义在 `README.md` 中(因为它是对外展示的入口),各 skill 的规格定义在其 `SKILL.md` 中。此索引服务于能解析这些文件的 Agent。

> 规格文档本身建议保持精简——详细的交互细节、视觉稿,如果存在于仓库外(设计工具等),至少在这里留一句"现状摘要 + 在哪能找到完整版",避免 agent 完全看不到这部分上下文。
